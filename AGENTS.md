# AGENTS.md

## Philosophy
- Keep changes additive
- No breaking changes without reason
- No unnecessary abstractions
- Favor readability over cleverness

## Development Rules
- No dependencies unless absolutely required
- Prefer built-in Node APIs
- Keep functions small and explicit
- Avoid “framework-like” patterns

## Testing
- Add regression tests for bugs
- Keep tests minimal and fast
- Focus on behavior, not implementation

## Logging Principles
- Logs must be structured
- Logs must be useful without configuration
- Avoid noise

## Release Strategy
- Ship small, frequent versions
- Maintain backwards compatibility
